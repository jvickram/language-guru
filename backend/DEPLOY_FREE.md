# Free Backend Hosting (Render)

This backend can be deployed for free using Render.

## 1) Push your latest code

Render deploys from your Git repo, so make sure your branch is pushed.

## 2) Deploy using Blueprint (recommended)

1. Open Render Dashboard.
2. Choose New + -> Blueprint.
3. Connect your GitHub repo.
4. Render auto-detects [render.yaml](../render.yaml).
5. Click Apply.

The service will be created as `language-guru-backend` with:
- Root directory: `backend`
- Build command: `npm install --omit=dev --no-audit --no-fund`
- Start command: `npm start`
- Health endpoint: `/api/health`

Note: this repo currently ignores lockfiles globally. If you want deterministic installs with `npm ci`, commit `backend/package-lock.json` first.

## 3) Environment variables used

Configured in [render.yaml](../render.yaml):
- `NODE_ENV=production`
- `DATA_STORE=memory`
- `RATE_LIMIT_PER_MINUTE=300`
- `ALLOWED_ORIGINS=*`
- `JWT_SECRET` auto-generated by Render

## 4) Verify deployment

After deploy completes, test:

```bash
curl https://<your-render-domain>/api/health
```

Expected response shape:

```json
{
  "status": "ok",
  "service": "language-guru-api",
  "env": "production"
}
```

## 5) Optional: persistent data (still free)

Current default is `DATA_STORE=memory`, which resets on restart.

If you want persistence:
1. Create a free MongoDB Atlas cluster.
2. In Render service env vars, set:
   - `DATA_STORE=mongo`
   - `MONGO_URI=<your-atlas-connection-string>`

## 6) Point the mobile app to hosted backend

Once deployed, use your Render URL in API config.

Edit [src/config/api.js](../frontend/src/config/api.js) and replace host logic with your hosted base URL when needed.
