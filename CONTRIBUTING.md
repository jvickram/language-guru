# Contributing to Language Guru

Thank you for considering contributing to Language Guru! We welcome contributions from everyone. Please follow the guidelines below to help us maintain a high-quality project.

## Contribution Guidelines
- Please ensure that your contributions are well-tested and that tests are added for new features or bug fixes.
- Adhere to our coding standards and style. You can check our style guide in the [Style Guide](LINK_TO_STYLE_GUIDE).

## Branch Naming Conventions
- Use the following format for your branch names:
  - `feature/description` for new features (e.g., `feature/add-login`)
  - `bugfix/description` for bug fixes (e.g., `bugfix/fix-typo`)
  - `hotfix/description` for urgent fixes (e.g., `hotfix/urgent-issue`)

## Commit Message Format
- Commit messages should be clear and descriptive. Use the following format:
  - `type(scope): subject`
  - Types can include `feat`, `fix`, `docs`, `style`, `refactor`, `test`, etc.
  - Example: `feat(login): add login functionality`

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Make your changes and ensure they are well-documented in your commit messages.
3. Push your branch to your forked repository.
4. Open a pull request to the main repository and ensure you follow the template.
5. Wait for a review from maintainers before merging your changes.

Happy contributing!
